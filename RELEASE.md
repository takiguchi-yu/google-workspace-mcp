## 【日々の運用フロー】リリースの手順

リリース作業は**ターミナルで以下のコマンドを2つ打つだけ**になります。ファイルの書き換えやタグの手動作成は一切不要です。

### Step 1: バージョンの引き上げ

ターミナルで `npm version` コマンドを実行します。
（※ `patch` はバグ修正等の小規模アップデート、`minor` は機能追加、`major` は破壊的変更の際に使い分けます）

```bash
# 例: 1.0.0 から 1.0.1 にアップデートする場合
npm version patch

```

> **💡 裏側で起きていること:**
>
> 1. `package.json` のバージョンが自動更新されます。
> 2. `sync-version.js` が走り、`server.json` のバージョンも同期されます。
> 3. 更新されたファイルが自動でGitコミットされ、`v1.0.1` というタグが自動で打たれます。

### Step 2: GitHubへのプッシュ（リリース実行）

自動作成されたコミットとタグを、GitHubの `main` ブランチにプッシュします。

```bash
git push origin main && git push origin --tags

```

> **💡 裏側で起きていること:**
> タグ（例: `v1.0.1`）がプッシュされたことを検知して、設定した GitHub Actions が自動で走り出します。
> これにより、**「Docker Hub へのイメージ push」「npm へのパッケージ公開」「MCP Registry へのサーバー登録」**がすべて全自動で完了します。
